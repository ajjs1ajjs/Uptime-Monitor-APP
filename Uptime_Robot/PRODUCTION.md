## 🚀 Швидкий старт для продакшн

```batch
:: 1. Перейдіть в папку UptimeMonitor_EXE
cd C:\UptimeMonitor

:: 2. Встановіть як службу Windows (від Адміністратора)
install_task.bat

:: 3. Відкрийте в браузері
start http://localhost:8080
```

## 📁 Структура файлів

```
UptimeMonitor_EXE/
├── UptimeMonitor.exe          # Головна програма
├── install_task.bat           # Встановлення
├── uninstall_task.bat         # Видалення

├── README.txt                 # Інструкція
├── port.txt                   # Файл з портом (8080)
├── sites.db                   # База даних (створиться автоматично)
└── uptime_monitor.log         # Лог файл
```

## 🔧 Команди управління

```batch
:: Перевірка статусу
schtasks /query /tn "UptimeMonitor"

:: Запуск
schtasks /run /tn "UptimeMonitor"

:: Зупинка
schtasks /end /tn "UptimeMonitor"

:: Перевірка процесу
tasklist | findstr UptimeMonitor

:: Перегляд логу
type uptime_monitor.log
```

## 📝 Важливі нотатки

1. **Папка встановлення**: Виберіть постійне місце (не OneDrive/Dropbox)
2. **Права доступу**: Потрібні права адміністратора тільки для install/uninstall
3. **Порт**: За замовчуванням 8080, можна змінити в port.txt
4. **База даних**: Автоматично створюється sites.db в папці з програмою
5. **Логування**: Помилки пишуться в uptime_monitor.log

## 🆘 Якщо щось не працює

1. **Не заходить на сайт**:
   - `schtasks /query /tn "UptimeMonitor"` - перевірити статус
   - `type uptime_monitor.log` - перевірити помилки
   - Переконатися що порт 8080 вільний

2. **Дані не зберігаються**:
   - Переконатися що запускаєте через install_task.bat
   - Перевірити права на запис в папку
   - Перевірити що sites.db створюється в папці з програмою

3. **Не запускається**:
   - Запустити test.bat для перевірки
   - Перевірити лог файл
   - Переконатися що порт не зайнятий
