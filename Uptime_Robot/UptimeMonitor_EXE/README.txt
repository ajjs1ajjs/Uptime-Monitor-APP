========================================
UPTIME MONITOR - WINDOWS PRODUCTION EDITION
========================================

Версія з автозапуском через Task Scheduler
Для продакшн використання

========================================
ШВИДКИЙ СТАРТ (Продакшн)
========================================

1. Розпакуйте папку UptimeMonitor_EXE в постійне місце
   Наприклад: C:\UptimeMonitor або D:\UptimeMonitor

2. Запустіть install_task.bat від імені Адміністратора

3. Відкрийте браузер: http://localhost:8080

4. Налаштуйте сайти для моніторингу

5. Дані зберігатимуться автоматично!

========================================
ПЕРЕВІРКА РОБОТИ
========================================

Перевірити статус задачі:
  schtasks /query /tn "UptimeMonitor"

Перевірити чи працює процес:
  tasklist | findstr UptimeMonitor

Перевірити лог:
  type uptime_monitor.log

Відкрити в браузері:
  http://localhost:8080

========================================
УПРАВЛІННЯ
========================================

Зупинити:     schtasks /end /tn "UptimeMonitor"
Запустити:    schtasks /run /tn "UptimeMonitor"
Статус:       schtasks /query /tn "UptimeMonitor"
Відкрити Task Scheduler: taskschd.msc

========================================
ФАЙЛИ
========================================

UptimeMonitor.exe     - головна програма (веб-сервер + моніторинг)
install_task.bat      - встановлення автозапуску (рекомендовано)
uninstall_task.bat    - видалення
start.bat            - ручний запуск (для тестування)
test.bat             - тестовий запуск з консоллю
UptimeMonitor.vbs    - скрипт для фонового запуску
ssl_checker.py       - модуль перевірки SSL
sites.db             - база даних (створюється автоматично)
port.txt             - файл з портом (8080 за замовчуванням)
uptime_monitor.log   - лог файл з помилками

========================================
ВАЖЛИВО - ЗБЕРЕЖЕННЯ ДАНИХ
========================================

✅ База даних (sites.db) створюється автоматично в папці з програмою
✅ Всі налаштування зберігаються при вимкненні
✅ Сайти залишаться після перезавантаження
✅ Налаштування сповіщень зберігаються
✅ Історія статусів сайтів зберігається

========================================
ВИДАЛЕННЯ
========================================

Запустіть від імені Адміністратора:
  uninstall_task.bat

========================================
НАЛАШТУВАННЯ ПОРТУ
========================================

За замовчуванням: 8080

Щоб змінити порт:
1. Відкрийте port.txt в текстовому редакторі
2. Змініть число на потрібний порт (наприклад, 8000)
3. Перезапустіть програму

========================================
ПРОБЛЕМИ ТА РІШЕННЯ
========================================

❌ "Не можу відкрити сайт"
→ Перевірте: schtasks /query /tn "UptimeMonitor"
→ Перевірте лог: type uptime_monitor.log

❌ "Дані не зберігаються"
→ Перевірте що запускаєте через install_task.bat
→ Перевірте права доступу до папки

❌ "Порт зайнятий"
→ Змініть порт в port.txt
→ або зупиніть іншу програму на цьому порті

========================================
СИСТЕМНІ ВИМОГИ
========================================

- Windows 10/11 або Windows Server 2016+
- Права адміністратора для встановлення
- Порт 8080 (або інший вказаний) вільний
- Доступ до інтернету для перевірки сайтів

========================================
ПІДТРИМКА
========================================

При виникненні проблем:

2. Запустіть test.bat для тестування
