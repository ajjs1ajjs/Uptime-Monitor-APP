# Uptime Monitor - Виправлення та покращення

## Що було виправлено

### 1. Проблема з Task Scheduler (не заходить на веб-адресу)
**Причина:** VBS скрипт використовував `%~dp0` який не працює в VBS, і не встановлював робочу директорію.

**Виправлення:**
- Оновлено `UptimeMonitor.vbs` - тепер коректно визначає шлях до папки та встановлює робочу директорію
- Оновлено `install_task.bat` - тепер створює задачу через XML з правильною робочою директорією

### 2. Проблема зі збереженням даних (дані не зберігаються)
**Причина:** База даних `sites.db` створювалась в поточній директорії, яка відрізнялась при запуску з Task Scheduler.

**Виправлення:**
- Оновлено `main.py` - тепер APP_DIR коректно визначається як папка де знаходиться EXE/скрипт
- База даних завжди створюється в папці з програмою незалежно від робочої директорії

### 3. Проблема з дизайном SSL сертифікатів
**Виправлення:**
- Зменшено розмір карток SSL сертифікатів
- Зменшено відступи та шрифти
- Зроблено компактніший grid для SSL (250px замість 350px)
- Сайти теж трохи зменшені (280px замість 350px)

## Як перезібрати EXE

### Спосіб 1: Через build_exe.bat (рекомендовано)
1. Відкрийте командний рядок в папці `D:\Project\Uptime_Robot`
2. Запустіть: `build_exe.bat`
3. Скрипт автоматично:
   - Встановить залежності
   - Збере EXE через PyInstaller
   - Скопіює файли в папку `UptimeMonitor_EXE`

### Спосіб 2: Вручну
```bash
cd D:\Project\Uptime_Robot
pip install pyinstaller fastapi uvicorn aiohttp pydantic pywin32
pyinstaller main.spec --clean --onefile
```

## Як встановити

### Варіант 1: Task Scheduler (рекомендовано) - працює у фоні
1. Перейдіть в папку `UptimeMonitor_EXE`
2. Запустіть `install_task.bat` від імені Адміністратора
3. Відкрийте браузер: http://localhost:8080
4. Дані зберігатимуться автоматично

### Варіант 2: Ручний запуск - для тестування
1. Перейдіть в папку `UptimeMonitor_EXE`
2. Запустіть `start.bat`
3. При закритті вікна програма зупиниться

## Файли які оновлено

- `main.py` - виправлено шлях до бази даних
- `main.spec` - оновлено hidden imports
- `UptimeMonitor_EXE/UptimeMonitor.vbs` - виправлено шляхи
- `UptimeMonitor_EXE/install_task.bat` - оновлено створення задачі
- `UptimeMonitor_EXE/start.bat` - додано cd до папки
- `UptimeMonitor_EXE/test.bat` - додано cd до папки
- `build_exe.bat` - новий файл для збірки

## Перевірка роботи

Після встановлення через Task Scheduler:
1. Перевірте статус: `schtasks /query /tn "UptimeMonitor"`
2. Перевірте процес: `tasklist | findstr UptimeMonitor`
3. Відкрийте http://localhost:8080
4. Додайте сайт - він повинен залишитись після перезапуску

## Як видалити

Запустіть від імені Адміністратора:
`UptimeMonitor_EXE/uninstall_task.bat`
